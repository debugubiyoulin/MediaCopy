# 媒体拷贝助手 - MediaCopy
这个软件旨在解决校园学生媒体和自媒体的一些问题，且集成了很多小功能。

* * *

### 事情是这样的……

在我们学校的运动会结束后，现任会长（驼狗）跟我讲，素材拷贝到最后都不知道谁是谁的了，所以想要一个 **可以批量加前缀的应用**。

我说 **“那敢情好啊，我正好有点编程基础，现在AI又这么厉害，这活我接了”**

## *德霖踏上了循环*

因为 **AI** 确实很强大，我只用了1个小时不到就完成了 V0.1 版本的制作和封装……

~~但是这个软件他太不优雅了，看着就像一个……命令窗（好吧，他其实就是一个命令窗）~~

因为使用BAT批处理文件编写，再通过软件直接封装的EXE文件，所以不仅外观奔放，而且功能太简单了

**反正我不能接受**

## *德霖进行了第一次重构*

这便是 V1.0 诞生的原因，纯粹是因为我觉得第一个版本不好看，毕竟在熟悉了图形化界面应用的当代，没有图像化界面还是很艰难的

也就是在这时，我发现了 **C#** 这个语言太好用了，毕竟自带图形化界面的库，我用以前的DevC++捣鼓了半天，不如C#一句话……

**不出意外那就要出意**

我自己对程序的功能不满意了

*这个软件也太差劲了吧，就这么一个功能，竟然有2M，我奶奶写的程序都比这个好（bushi*

反正说一千道一万就是一句话 

### “我要重构”

## *德霖进行了第二次重构*

这次重构我的想法是创建一个好看，简洁，功能强的界面，本质其实区别不大。

**当然最主要的是现实一种AI开发的方式**

第一步当然是通过画图理解并拆分自己的需求，这和通常软件开发逻辑应该相似，没有明确的目标是写不了代码的

第二步是写提示词，这是通过AI写代码很重要的一步
